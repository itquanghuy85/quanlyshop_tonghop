# HULUCA Shop Manager — Context for Claude

## Thông tin cơ bản
- App: HULUCA Shop Manager (`com.huluca.shopmanager`) v10.1.0
- Stack: Flutter + Firebase (Firestore + Auth + FCM + Storage)
- DB: SQLite (`repair_shop_v22.db`, v80, 30+ bảng, `db_helper.dart` 7024 dòng)
- Sync: Hybrid offline-first — SQLite local + Firestore cloud
- Quy mô: ~215 files, ~95,000 dòng, 43 services, 80+ views
- Ngôn ngữ: Tiếng Việt / English (i18n ~2400 keys)
- Chủ app: không biết lập trình, xây dựng hoàn toàn bằng AI (GitHub Copilot trong VS Code)

## Kiến trúc
```
UI (Flutter) → Service Layer → SQLite ↔ Firestore
```
- **Service-first**: Widget không bao giờ gọi Firebase trực tiếp
- **EventBus**: broadcast stream cho cross-widget communication (repairs_changed, sales_changed, debts_changed, expenses_changed, products_changed, shopChanged)
- **SyncOrchestrator**: offline queue 19 entity types, max 3 retry, auto-sync khi có mạng
- **Conflict resolution**: local `isSynced=false` → REJECT cloud, giữ local changes

## Payment System (quan trọng)
- Mọi thanh toán BẮT BUỘC qua `PaymentIntentService` — có `PaymentBlocker.block()` để chặn bypass
- Flow: Business module → PaymentIntent → MoneyValidationService → MoneyTransactionService → FinancialActivityService
- 2 Firestore transaction atomic: `executeDebtPaymentTransaction()` và `executeSaleTransaction()` (chống race condition)

## Security
- AES-256-CBC mã hóa 23 trường nhạy cảm trước khi lên cloud (key = SHA256(shopId + masterSecret))
- Custom Claims JWT: role + shopId + isSuperAdmin
- Firestore rules 1081 dòng, audit log IMMUTABLE
- Super admin: admin@huluca.com

## Phân quyền
- superAdmin > owner > manager > employee/technician > user
- Owner: xem giá vốn, lợi nhuận, tài chính
- Manager: không xem revenue/expense/debt
- Employee/Technician: chỉ thao tác cơ bản

## Multi-tenant / Multi-shop
- Mọi document có `shopId`, enforce qua Firestore rules + Custom Claims
- Owner có thể quản lý nhiều shop, switch shop xóa SQLite local và reinit sync

## Multi-industry
- electronics (mặc định): repair + IMEI + warranty
- food: expiry + batch
- fashion: variants (size/color)
- general: basic

## Models chính
- `Repair`: 4-step status (1=nhận, 2=đang sửa, 3=xong, 4=giao)
- `SaleOrder`: finalPrice=totalPrice-discount, có installment fields (downPayment, settlementAmount, loanAmount, bankName)
- `Product`: type = DIEN_THOAI / LINH_KIEN / PHU_KIEN
- `Debt`: type = CUSTOMER_OWES / SHOP_OWES / OTHER_CUSTOMER_OWES / OTHER_SHOP_OWES
- `PaymentIntent`: 17 types, status = pending→completed/cancelled/failed
